API de Siretisation

API REST pour le matching d'etablissements avec le registre SIRENE (INSEE). Retrouvez automatiquement le SIREN/SIRET d'entreprises a partir de leur nom, adresse et autres informations.

URL de base : https://srv1.net-it-be.com:3001/api/v1
Authentification : Header X-API-Key: votre_cle ou parametre ?api_key=votre_cle
Format : JSON (Content-Type: application/json)

🔑 Authentification

Toutes les routes /api/v1/siretisation/* et /api/v1/taches/* requierent une cle API. Les routes /admin/* et /api/v1/admin/* sont en acces libre.

MethodeExemple

Query parameter?api_key=sk_siret_abc123...

🔍 Siretisation
POST /api/v1/siretisation/unitaire Recherche synchrone (1 etablissement) Cle API ▶

Recherche un seul etablissement dans le registre SIRENE. La reponse est synchrone et contient directement le resultat du matching.

Parametres (body JSON)
ParametreTypeRequisDescription
nomstringouiNom / raison sociale de l'etablissement
adressestringnonAdresse complete
code_postalstringnonCode postal (5 chiffres)
villestringnonNom de la ville
telephonestringnonNumero de telephone
siretstringnonSIRET connu (verification)
Codes de reponse
200 OK 400 Bad Request 401 Unauthorized 500 Server Error
Exemple de requete
POST /api/v1/siretisation/unitaire Content-Type: application/json X-API-Key: sk_siret_votre_cle { "nom": "Boulangerie Martin", "adresse": "12 rue de la Paix", "code_postal": "75002", "ville": "Paris" }
Exemple de reponse
{ "statut": "trouve", "confiance": 0.92, "resultat": { "siren": "123456789", "siret": "12345678900010", "denomination": "BOULANGERIE MARTIN SARL", "adresse": "12 RUE DE LA PAIX 75002 PARIS", "code_naf": "10.71C", "categorie_juridique": "5710" } }

POST /api/v1/siretisation/batch/google-business Batch asynchrone (Google Business) Cle API ▶

Lance un traitement batch asynchrone en interrogeant Google Business pour enrichir les donnees avant le matching SIRENE. Retourne un ID de tache pour suivre la progression.

Parametres (body JSON)
ParametreTypeRequisDescription
etablissementsarrayouiTableau d'objets {nom, adresse, code_postal, ville, ...}
Codes de reponse
201 Created 400 Bad Request 401 Unauthorized
Exemple de reponse
{ "tache_id": 42, "type": "batch_google", "statut": "en_attente", "total_fiches": 150, "message": "Tache creee avec succes. Utilisez GET /api/v1/taches/42 pour suivre la progression." }
POST /api/v1/siretisation/batch/fichier Batch asynchrone (upload fichier) Cle API ▶

Upload un fichier CSV/XLSX contenant une liste d'etablissements et lance le matching batch asynchrone. Taille max : 50 Mo.

Parametres (multipart/form-data)
ParametreTypeRequisDescription
fichierfileouiFichier CSV ou XLSX (max 50 Mo)
Codes de reponse
201 Created 400 Bad Request 401 Unauthorized
Exemple avec cURL
curl -X POST https://srv1.net-it-be.com:3001/api/v1/siretisation/batch/fichier \ -H "X-API-Key: sk_siret_votre_cle" \ -F "fichier=@etablissements.csv"

GET /api/v1/taches/:id/resultats Resultats detailles d'une tache Cle API ▶

Retourne la liste complete des resultats de matching pour une tache terminee, avec les SIREN/SIRET trouves et les scores de confiance.

Parametres URL
ParametreTypeDescription
:idintegerID de la tache
Codes de reponse
200 OK 401 Unauthorized 500 Server Error
GET /api/v1/taches/:id/resultats/export Export XLSX des resultats Cle API ▶

Telecharge les resultats de matching sous forme de fichier Excel (.xlsx). Ideal pour l'integration dans des outils bureautiques.

Reponse

Retourne un fichier binaire application/vnd.openxmlformats-officedocument.spreadsheetml.sheet avec le header Content-Disposition: attachment.

Exemple avec cURL
curl -H "X-API-Key: sk_siret_votre_cle" \ https://srv1.net-it-be.com:3001/api/v1/taches/42/resultats/export \ -o resultats_tache_42.xlsx
